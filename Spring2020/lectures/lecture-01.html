Elements of Data Science

This semester we will follow a new curriculum, called Elements of Data Science, that I am developing along with a group at Harvard.

https://allendowney.github.io/ElementsOfDataScience/

It includes a series of Jupyter notebooks that start with basic Python and work through the process of exploratory data analysis.

You have two options for working with the notebooks:

1) Run them on Google Colab and store them in Google Drive.

2) Run them in your own Jupyter environment and store them on GitHub.

Either way, what you turn in will be a link to a notebook we can run on Colab.

Colab Mental Model

Notes from notebook 1

Numeric types: integers are for counting; floats are for measuring.

Apparently this came up in ModSim last semester

Let this be an early warning about false precision

Integers are exact, but they have a more limited range.

Floating-point numbers have wider range, but they are approximate (or you should assume they are)

In general, you should never compare floating-point numbers for equality.

>>> 0.1 + 0.1 + 0.1 == 0.3
False

Use np.isclose (read the docs)

If you are not familiar with NumPy, that's one of the things you will learn in this class.

The first notebook is also meant to remind you about exp() and log(), because we are going to use them a lot.

And finally, there's just a little algebra.  There's not much calculus in this class, but we will dust off the algebra from time to time.

The Art of Statistics

The textbook for this class is Spiegelhalter, The Art of Statistics.

It's available in various formats for a reasonable price, and we have two copies on reserve in the library.

I think it's a good, high-level description of the topics we'll cover in this class.

It's written at a non-technical level, so we'll use it to introduce ideas, then we'll get into the details, both computational and mathematical.

The reading quizzes are intended to provide:
  • Incentive
  • Active reading
  • Spaced repetition
  • Feedback for you (are you understanding?)
  • Signal for me (what's clear, what's not?)

You probably know about strings.

You might not know about pd.Timestamp and pd.Timedelta.

There's are differences between Timestamp and Timedelta that are subtle, but important.

And they imply an Algebra of Time (that is, a set of operations you can perform with these values).

Data Science 2020
Data Science 2020 intro

The view I present in this talk is generally optimistic.  I think the increasing availability of data will tend to make our understanding of the world better, which will help us make better decisions.

But the history of the last few years seems calculated to contradict me.

This contradiction will be a recurring theme of this class, something I hope we will discuss as opportunities arise.  I don't know if we will resolve it, but maybe by the end of the semester we will have a better understanding of it.

Software

You are welcome to use Colab for the EDS notebooks; in fact, I prefer it.

But later in the semester you might prefer to work on your laptop.  

The CAs and I can help, but here's some info to get started.

1) You will need NumPy, SciPy, Pandas and some other packages.  If you are not already using Anaconda, which is a Python distribution that provides these packages and many more, I strongly recommend it.

2) You should use Python 3, ideally 3.6 or 3.7.  If you are using an older version, consider starting with a fresh Anaconda installation.  Otherwise, to update the packages you have installed, run

conda update --all

But don't do it during class!
